การป้องกันภัยทุจริตทางไซเบอร์ (Cybersecurity)

💀 ภัยคุกคาม 🗂️ จัดชั้นความลับ 🔑 รหัสผ่าน 📧 ฟิชชิง 🚨 ข้อมูลรั่วไหล 🏦 มาตรการ ธ.ก.ส. 📝 เทคนิคสอบ
💀
ภัยคุกคามทางไซเบอร์ที่ต้องรู้
ประเภทหลักของภัยคุกคามที่ธนาคารเผชิญ

🎣 Social Engineering / Phishing

หลอกให้คลิกลิงก์หรือเปิดเผยข้อมูล ผ่านอีเมล LINE โทรศัพท์ SMS ปลอมเป็นคนที่น่าเชื่อถือ — ภัยที่พบบ่อยที่สุด

🦠 Malware / Ransomware

ซอฟต์แวร์ที่เป็นอันตราย เข้ารหัสไฟล์แล้วเรียกค่าไถ่ หรือขโมยข้อมูล — มักมาจากการดาวน์โหลดหรือคลิกลิงก์ผิด

💉 Data Breach / ข้อมูลรั่วไหล

ข้อมูลลูกค้าถูกเข้าถึงโดยไม่ได้รับอนุญาต อาจเกิดจากแฮกเกอร์ภายนอก หรือพนักงานภายในที่ทุจริตหรือประมาท

🔒 Account Takeover / บัญชีม้า

ยึดบัญชีลูกค้าด้วยข้อมูลที่ขโมยมา หรือการสร้างบัญชีปลอมเพื่อทำธุรกรรมผิดกฎหมาย

🗂️
การจัดชั้นความลับข้อมูล (Data Classification)
4 ระดับ — ต้องรู้ว่าข้อมูลแต่ละชนิดอยู่ระดับไหน
🔴

ลับที่สุด

ข้อมูลลูกค้า, รหัสผ่านระบบ, แผนยุทธศาสตร์, ข้อมูลการเงินสำคัญ

🟠

ลับ

ข้อมูลพนักงาน, รายงานภายใน, ข้อมูลธุรกรรม, ข้อมูลสัญญา

🔵

สำหรับใช้ภายใน

นโยบาย ระเบียบ คู่มือ ประกาศภายใน ไม่เปิดเผยต่อสาธารณะ

⚪

สาธารณะ

ข้อมูลที่เปิดเผยต่อสาธารณะได้ เช่น ข้อมูลผลิตภัณฑ์ ประกาศทั่วไป

ส่งข้อมูลลับภายในธนาคาร: ต้องใช้ช่องทางที่ธนาคารกำหนด เช่น ระบบสารบรรณ อีเมลองค์กร — ห้ามส่งทาง LINE กลุ่มส่วนงาน
🔑
การตั้งรหัสผ่านที่รัดกุม
แนวปฏิบัติที่ดีด้านรหัสผ่าน

✅ ควรทำ

  • รหัสผ่านอย่างน้อย 12 ตัวอักษร
  • ผสมตัวอักษรพิมพ์ใหญ่-เล็ก ตัวเลข สัญลักษณ์
  • ใช้รหัสผ่านต่างกันในแต่ละบัญชี
  • เปลี่ยนรหัสผ่านเป็นระยะ
  • ใช้ Multi-Factor Authentication (MFA)
  • ใช้ Password Manager ที่น่าเชื่อถือ

❌ ห้ามทำ

  • ใช้ข้อมูลส่วนตัว เช่น วันเกิด ชื่อ
  • ใช้รหัสผ่านเดิมซ้ำหลายบัญชี
  • บอกรหัสผ่านให้ผู้อื่น แม้จะเป็นผู้บังคับบัญชา
  • จดรหัสผ่านทิ้งไว้บนโต๊ะหรือในไฟล์ที่เข้าถึงได้ง่าย
  • ใช้รหัสง่ายๆ เช่น 123456, password
📧
วิธีเช็กอีเมลหลอก (Phishing Detection)
สัญญาณเตือนที่ต้องสังเกต
สัญญาณวิธีตรวจสอบ
📧 ที่อยู่อีเมลผู้ส่งผิดปกติ ดูชื่อโดเมน เช่น @baac-th.net แทน @baac.or.th = น่าสงสัย
🚨 ข้อความเร่งด่วน สร้างความกลัว "บัญชีของคุณจะถูกระงับ! คลิกทันที" = กลยุทธ์หลอกลวง
🔗 ลิงก์ที่ไม่ตรงกับเนื้อหา Hover เมาส์ดู URL จริงก่อนคลิก
📎 ไฟล์แนบที่ไม่ได้คาดหวัง อย่าเปิดไฟล์ .exe, .zip จากแหล่งที่ไม่รู้จัก
📝 ภาษาและไวยากรณ์ผิดปกติ อีเมลทางการมักเขียนถูกต้อง ไม่สะกดผิด
💰 ขอข้อมูลส่วนตัวหรือรหัสผ่าน ธนาคารที่น่าเชื่อถือไม่มีวันขอรหัสผ่านทางอีเมล
🚨
สิ่งที่ควรทำเมื่อข้อมูลรั่วไหล
4 ขั้นตอนหลักในการตอบสนองเหตุการณ์
ขั้นตอนการดำเนินการระยะเวลา
1. ตรวจจับและระงับ แจ้งฝ่ายไอที ตัดการเชื่อมต่อระบบที่ถูกโจมตี เปลี่ยนรหัสผ่านทันที ทันที
2. ประเมินผลกระทบ ระบุว่าข้อมูลใดรั่วไหล จำนวนเท่าไร กระทบใครบ้าง ภายใน 24 ชม.
3. แจ้งหน่วยงานที่เกี่ยวข้อง แจ้ง PDPA Office ภายใน 72 ชั่วโมง แจ้ง ธปท. และหน่วยงานกำกับดูแล ภายใน 72 ชม.
4. ฟื้นฟูและป้องกัน แก้ไขช่องโหว่ ทบทวนมาตรการ อบรมพนักงาน จัดทำรายงานเหตุการณ์ หลังเหตุการณ์
🏦
มาตรการ Cybersecurity ของ ธ.ก.ส. ปี 2569
ตามแผนวิสาหกิจและ พ.ร.ก. อาชญากรรมทางเทคโนโลยี

🎯 เป้าหมาย Cyber Security 100%

ตรวจจับและตอบสนองภัยคุกคามภายใน SLA (Service Level Agreement) 100% — เปลี่ยนจากการป้องกันเชิงรับ เป็น ระบบตรวจจับอัตโนมัติ

🤖 AI ตรวจจับภัยคุกคาม

ใช้ AI และ Machine Learning ตรวจจับพฤติกรรมผิดปกติแบบ Real-time แทนการรอให้เกิดเหตุก่อน

📱 ป้องกันอาชญากรรมทางเทคโนโลยี

ตาม พ.ร.ก. มาตรการป้องกันและปราบปรามอาชญากรรมทางเทคโนโลยี พ.ศ. 2568 ธนาคารต้องพัฒนาระบบตรวจบัญชีต้องสงสัยเพื่อรายงาน ธปท.

🔍 ระบบตรวจจับบัญชีม้า

พัฒนาระบบ AI ตรวจสอบพฤติกรรมธุรกรรมผิดปกติ คัดกรองบัญชีที่น่าสงสัย รายงานต่อหน่วยงานที่กำกับดูแล

บริบทสำคัญ: ปี 2567 มีผู้เสียหายจากอาชญากรรมไซเบอร์ลดลงมากจากมาตรการ ธปท. แต่ยังต้องพัฒนาอย่างต่อเนื่อง — ในยุค Generative AI ภัยคุกคามมาเร็วกว่ามนุษย์ตรวจสอบได้

📝 เทคนิคทำข้อสอบ — Cybersecurity

① ห้ามส่งข้อมูลลับทาง LINE — ต้องใช้ช่องทางที่ธนาคารกำหนด (ระบบสารบรรณ/อีเมลองค์กร)
② ธนาคารจะไม่ขอรหัสผ่านทางอีเมล/โทรศัพท์ — ถ้าถูกขอ = Phishing/Social Engineering
③ แจ้งเหตุรั่วไหล PDPA ภายใน 72 ชั่วโมง — เชื่อมกับข้อ 9 PDPA ด้วย
④ Cybersecurity ปี 2569 = ระบบตรวจจับอัตโนมัติ ไม่ใช่แค่ Firewall เพราะภัยใหม่มาเร็วเกินกว่ามนุษย์รับมือได้
⑤ MFA (Multi-Factor Authentication) = การยืนยันตัวตนหลายขั้นตอน เพิ่มความปลอดภัยกว่าแค่รหัสผ่าน
⑥ ข้อมูลลับที่สุด = ข้อมูลลูกค้า รหัสผ่านระบบ — อย่าเปิดเผย แม้กับเพื่อนร่วมงาน
